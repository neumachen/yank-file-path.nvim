1. INTRODUCTION                                     *yank-file-path-introduction*

yank-file-path.nvim is a Neovim plugin that provides convenient commands to
copy file paths to the system clipboard. It supports various path formats
including relative paths, absolute paths, and file names only.

The plugin also supports copying multiple buffer paths at once with
customizable separators. The plugin is built with a modular architecture
for maintainability and extensibility.

2. INSTALLATION                                     *yank-file-path-installation*

Using lazy.nvim: >
    {
        "neumachen/yank-file-path.nvim",
        config = function()
            require("yank-file-path").setup({
                -- Optional: customize root markers
                root_markers = { ".git", "package.json", "go.mod" }
            })
        end,
    }
<

Or with minimal configuration: >
    {
        "neumachen/yank-file-path.nvim",
        -- Plugin loads automatically with default settings

3. COMMANDS                                             *yank-file-path-commands*

The plugin provides the following commands:

                                                    *:YankRelativeFilePath*
:YankRelativeFilePath
    Copy the relative file path of the current buffer to the clipboard.
    The path is relative to the current working directory.

                                                    *:YankAbsoluteFilePath*
:YankAbsoluteFilePath
    Copy the absolute file path of the current buffer to the clipboard.

                                            *:YankRelativeFilePathFromHome*
:YankRelativeFilePathFromHome
    Copy the file path relative to the home directory (~) to the clipboard.

                                                           *:YankFileName*
:YankFileName
    Copy only the file name (without directory path) to the clipboard.

                                                 *:YankRootRelativeFilePath*
:YankRootRelativeFilePath
    Copy the file path relative to the project root directory to the clipboard.
    The root is determined by searching for root markers like .git, package.json, etc.

                                          *:YankRelativeFilePathWithLine*
:YankRelativeFilePathWithLine
    Copy the relative file path with current line number to the clipboard.
    Format: path/to/file.ext:42

                                          *:YankAbsoluteFilePathWithLine*
:YankAbsoluteFilePathWithLine
    Copy the absolute file path with current line number to the clipboard.
    Format: /full/path/to/file.ext:42

                                  *:YankRelativeFilePathFromHomeWithLine*
:YankRelativeFilePathFromHomeWithLine
    Copy the file path relative to home with current line number to the clipboard.
    Format: ~/path/to/file.ext:42

                                                   *:YankFileNameWithLine*
:YankFileNameWithLine
    Copy the filename with current line number to the clipboard.
    Format: file.ext:42

                                         *:YankRootRelativeFilePathWithLine*
:YankRootRelativeFilePathWithLine
    Copy the file path relative to project root with current line number.
    Format: src/file.ext:42

                                                   *:YankAllRelativeFilePaths*
:YankAllRelativeFilePaths [separator]
    Copy all relative file paths of loaded buffers to the clipboard.
    Optional separator can be specified (default is space).

                                                   *:YankAllAbsoluteFilePaths*
:YankAllAbsoluteFilePaths [separator]
    Copy all absolute file paths of loaded buffers to the clipboard.
    Optional separator can be specified (default is space).

                                           *:YankAllRelativeFilePathsFromHome*
:YankAllRelativeFilePathsFromHome [separator]
    Copy all file paths relative to home directory of loaded buffers.
    Optional separator can be specified (default is space).

                                                          *:YankAllFileNames*
:YankAllFileNames [separator]
    Copy all file names of loaded buffers to the clipboard.
    Optional separator can be specified (default is space).

                                             *:YankAllRootRelativeFilePaths*
:YankAllRootRelativeFilePaths [separator]
    Copy all file paths relative to project root of loaded buffers.
    Optional separator can be specified (default is space).

                                                           *:YankFilePath*
:YankFilePath [format] [options]
    Main command that supports all functionality with arguments.
    
    Format options:
    - relative    Copy relative file path (default)
    - absolute    Copy absolute file path  
    - filename    Copy filename only
    - home        Copy path relative to home directory (~)
    - root        Copy path relative to project root
    
    Options:
    - --line      Include current line number
    - --range     Include line range (works with visual selection)
    - --all       Copy paths from all loaded buffers
    - --separator=<sep>  Custom separator for multiple paths (default: space)
    - --root=<marker>    Specific root marker to search for (default: .git)
    
    Examples:
    :YankFilePath relative
    :YankFilePath absolute --line
    :YankFilePath filename --range
    :YankFilePath root --root=package.json
    :YankFilePath --all relative --separator="\n"

4. USAGE                                                   *yank-file-path-usage*

Basic usage:
1. Open a file in Neovim
2. Run any of the copy commands
3. The file path will be copied to your system clipboard
4. You'll see a notification confirming the copy operation

For multiple files:
1. Open multiple files in buffers
2. Use any of the "CopyAll*" commands
3. All loaded buffer paths will be copied with the specified separator

5. EXAMPLES                                             *yank-file-path-examples*

Basic command usage:

Copy current file's relative path: >
    :YankRelativeFilePath
<

Copy current file's absolute path: >
    :YankAbsoluteFilePath
<

Copy just the filename: >
    :YankFileName
<

Copy path relative to project root: >
    :YankRootRelativeFilePath
<

Copy relative path with line number: >
    :YankRelativeFilePathWithLine
<

Copy absolute path with line number: >
    :YankAbsoluteFilePathWithLine
<

Copy filename with line number: >
    :YankFileNameWithLine
<

Copy all relative paths separated by newlines: >
    :YankAllRelativeFilePaths \n
<

Copy all filenames separated by commas: >
    :YankAllFileNames ,
<

Copy all absolute paths separated by tabs: >
    :YankAllAbsoluteFilePaths \t
<

Special separators:
- `\n` - newline character
- `\t` - tab character
- Any other string is used as-is

                                                    *yank-file-path-keymaps*
Default Key Mappings:

The plugin automatically sets up the following key mappings using the pattern
`<leader>y` + format + modifiers:

Format letters:
- `r` = relative path
- `a` = absolute path  
- `f` = filename only
- `h` = home-relative path (~)
- `o` = root-relative path (project root)

Modifier letters:
- `l` = include line number
- `r` = include range (works with visual selection)
- `a` = all buffers
- `n` = newline separator (when combined with `a`)

Single file mappings: >
    <leader>yr   " Yank relative file path
    <leader>ya   " Yank absolute file path
    <leader>yf   " Yank filename only
    <leader>yh   " Yank home-relative path
    <leader>yo   " Yank root-relative path
<

Single file with line number: >
    <leader>yrl  " Yank relative path with line number
    <leader>yal  " Yank absolute path with line number
    <leader>yfl  " Yank filename with line number
    <leader>yhl  " Yank home-relative path with line number
    <leader>yol  " Yank root-relative path with line number
<

Single file with range (normal and visual mode): >
    <leader>yrr  " Yank relative path with range
    <leader>yar  " Yank absolute path with range
    <leader>yfr  " Yank filename with range
    <leader>yhr  " Yank home-relative path with range
    <leader>yor  " Yank root-relative path with range
<

All buffers (space-separated): >
    <leader>yra  " Yank all relative paths
    <leader>yaa  " Yank all absolute paths
    <leader>yfa  " Yank all filenames
    <leader>yha  " Yank all home-relative paths
    <leader>yoa  " Yank all root-relative paths
<

All buffers (newline-separated): >
    <leader>yran " Yank all relative paths (newlines)
    <leader>yaan " Yank all absolute paths (newlines)
    <leader>yfan " Yank all filenames (newlines)
    <leader>yhan " Yank all home-relative paths (newlines)
    <leader>yoan " Yank all root-relative paths (newlines)
<

Custom key mapping examples:

If you want to override the default mappings: >
    -- Disable default mappings in setup
    require("yank-file-path").setup({
        enable_default_mappings = false
    })
    
    -- Then set your own:
    vim.keymap.set('n', '<leader>yp', ':YankFilePath relative<CR>', 
                   { desc = 'Yank relative file path' })
    vim.keymap.set('n', '<leader>yP', ':YankFilePath absolute<CR>', 
                   { desc = 'Yank absolute file path' })
    vim.keymap.set('n', '<leader>yn', ':YankFilePath filename<CR>', 
                   { desc = 'Yank filename only' })
<

Which-key integration: >
    local wk = require("which-key")
    
    wk.register({
      y = {
        name = "Yank File Paths",
        r = { ":YankFilePath relative<CR>", "Relative path" },
        a = { ":YankFilePath absolute<CR>", "Absolute path" },
        f = { ":YankFilePath filename<CR>", "Filename only" },
        h = { ":YankFilePath home<CR>", "Home-relative path" },
        o = { ":YankFilePath root<CR>", "Root-relative path" },
      },
    }, { prefix = "<leader>" })
<

6. CONFIGURATION                                   *yank-file-path-configuration*

The plugin works out of the box without any configuration. All commands are
automatically available after the plugin is loaded.

                                                        *yank-file-path-setup*
Setup function:
>
    require("yank-file-path").setup({
        root_markers = { ".git", "package.json", "go.mod" },
        enable_default_mappings = true
    })
<

Configuration options:
- `root_markers` (table): List of files/directories to search for when
  determining project root. Default: { ".git" }
- `enable_default_mappings` (boolean): Whether to automatically set up the
  default key mappings. Default: true

Disable default mappings and set custom ones:
>
    require("yank-file-path").setup({
        enable_default_mappings = false
    })
    
    -- Set your own mappings
    vim.keymap.set('n', '<leader>yp', ':YankFilePath relative<CR>', 
                   { desc = 'Yank relative file path' })
    vim.keymap.set('n', '<leader>yP', ':YankFilePath absolute<CR>', 
                   { desc = 'Yank absolute file path' })
<

Manually set up default mappings later:
>
    require("yank-file-path").setup({
        enable_default_mappings = false
    })
    
    -- Later, conditionally set up mappings
    if some_condition then
        require("yank-file-path").setup_mappings()
    end
<

Runtime configuration:
>
    require("yank-file-path").set_root_markers({ ".git", "custom.toml" })
<

The plugin uses `vim.notify()` for notifications with INFO level for successful
operations and WARN level for warnings (e.g., when no buffers are found).
